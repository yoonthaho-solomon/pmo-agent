# AI Priority Dispatch Spec

작성일: 2026-06-15
대상: 기존 배차로직에 AI 우선순위 정렬을 추가하기 위한 개발 전달 문서

## 목적

이 문서는 기존 배차로직을 전면 교체하기 위한 문서가 아니다.

목표는 호출이 발생했을 때 기존 배차 후보군 안에서 콜카드 조건과 기사 운행 성향이 가장 잘 맞는 기사를 먼저 찾고, 그 기사에게 콜카드를 우선 발송할 수 있도록 AI 우선순위 점수를 제공하는 것이다.

핵심 문제는 요청수가 증가해도 기사가 수락하지 않으면 운행완료수가 늘지 않는다는 점이다. 따라서 `expired`, `canceled`, 무응답 가능성이 높은 콜을 줄이려면 콜 조건과 잘 맞는 기사에게 먼저 보내는 후보 정렬 로직이 필요하다.

## 적용 범위

### 포함

- 콜카드 정보를 22차원 콜카드 벡터로 변환
- 기사 운행 성향을 22차원 기사 벡터로 변환
- 콜카드 벡터와 기사 벡터의 코사인 유사도 계산
- 기존 배차 후보 기사 목록을 코사인 유사도 기준으로 우선 정렬
- 상위 기사에게 콜카드 우선 발송
- 미수락, 무응답, 만료 시 기존 배차 방식으로 fallback

### 제외

- 기존 배차 엔진 전체 재작성
- 최종 배차점수 가중치 확정
- ETA, 수락확률, 콜 위험도를 운영 점수에 즉시 혼합
- 실시간 위치 수집 시스템 설계 전체
- 기사앱 발송/응답 프로토콜 전체 변경

## 핵심 개념

호출이 발생하면 콜카드 정보로 22차원 벡터를 만든다.

기존 배차로직이 탐색 반경, 기사 상태, 지역 조건 등을 기준으로 후보 기사를 만든다.

후보 기사별로 저장된 22차원 기사 벡터를 조회한다.

콜카드 벡터와 기사 벡터의 코사인 유사도를 계산한다.

코사인 유사도가 높은 기사부터 콜카드를 우선 발송한다.

기사가 받지 않으면 다음 후보 또는 기존 순차/반경 확장 배차로 넘어간다.

## 추천 배차 흐름

```text
1. 승객 호출 발생
2. 콜카드 정보 생성
3. 콜카드 22D 벡터 생성
4. 기존 배차로직으로 후보 기사 탐색
5. 후보 기사 상태 필터
   - 온라인
   - 공차
   - 콜 수신 가능
   - 위치 최신성
   - 탐색 반경
6. 후보 기사 22D 벡터 조회
7. 콜카드 벡터와 기사 벡터 코사인 유사도 계산
8. 유사도 높은 순으로 후보 정렬
9. 상위 기사에게 콜카드 우선 발송
10. 미수락/무응답/만료 시 다음 후보 또는 기존 배차 fallback
```

## 콜카드 벡터 생성

콜카드는 현재 호출 조건을 one-hot 또는 점수형 값으로 변환한다.

필요 입력:

- `hour_slot`
- `weekday`
- `expected_distance`
- `expected_fare`
- `is_paid`
- `is_surge`
- `eta_distance`
- 향후 확장: `passenger_lat`, `passenger_lng`, `dest_lat`, `dest_lng`, 출발권역, 도착권역

거리 구간:

- 단거리: `0 < expected_distance <= 3000`
- 중거리: `3000 < expected_distance <= 8000`
- 장거리: `expected_distance > 8000`

요금 구간:

- 저요금: `0 < expected_fare <= 10000`
- 중간요금: `10000 < expected_fare <= 20000`
- 고요금: `expected_fare > 20000`

ETA/근접성:

```text
eta <= 150초: 1
eta >= 600초: 0
그 사이: 선형 감소
```

## 기사 벡터 생성

기사 벡터는 과거 운행/수락 패턴을 기반으로 만든다.

예시:

- 어떤 시간대에 주로 운행했는지
- 어떤 요일에 운행 비중이 높은지
- 단거리/중거리/장거리 중 어떤 콜을 많이 수행했는지
- 저요금/중간요금/고요금 콜 성향
- 유료콜/무료콜 성향
- 탄력/일반 상품 성향
- 평균 ETA 또는 근접 배차 성향

현재 테이블 기준:

- `driver_mbti`: 기사 22D 벡터 저장
- `callcard_mbti`: 콜카드 22D 계산에 필요한 원천 필드 저장
- `matching_scores`: 저장형 Top 10 추천 결과

## 코사인 유사도 계산

계산식:

```text
cosine_similarity = dot(call_vector, driver_vector) / (|call_vector| * |driver_vector|)
```

의미:

- 1에 가까울수록 콜 조건과 기사 성향이 잘 맞음
- 0에 가까울수록 성향 매칭이 약함

초기 운영 권장:

```text
ai_priority_score = cosine_similarity
```

초기에는 ETA, 콜 위험도, 예상 수락확률을 최종점수에 섞지 않는다.
먼저 코사인 유사도만으로 우선 발송 순서를 검증한다.

## 기존 배차로직과의 결합 방식

권장 방식은 기존 배차로직을 유지하고 후보 정렬 단계만 바꾸는 것이다.

```text
기존:
후보 기사 생성 -> 거리/순차/기존 기준 정렬 -> 콜 발송

제안:
후보 기사 생성 -> 22D 코사인 유사도 계산 -> 유사도 높은 순 정렬 -> 콜 발송
```

fallback:

```text
AI 우선 후보가 미수락/무응답/만료
-> 다음 유사도 후보에게 발송
-> 후보 소진 시 기존 반경 확장 또는 기존 순차 배차로 fallback
```

## 개발자 구현 요청 범위

1. 호출 발생 시 콜카드 정보 확보
2. 콜카드 22D 벡터 생성
3. 기존 탐색 반경 내 배차 후보 기사 목록 확보
4. 후보 기사별 `driver_mbti` 조회
5. 후보별 코사인 유사도 계산
6. 후보 리스트를 유사도 내림차순으로 정렬
7. 상위 기사부터 콜카드 발송
8. 발송/수락/거절/무응답/만료 이벤트 저장
9. 실패 시 기존 배차로직 fallback

AI 우선배차가 효과가 있는지 검증하려면 배차 결과 이벤트 저장이 필요하다.

저장 권장 이벤트:

- 후보 생성
- 콜카드 발송
- 기사 수락
- 기사 거절
- 무응답
- 만료
- 승객 취소
- 기사 취소
- 승차
- 하차/운행완료

현재 준비된 테이블/API 초안:

- `supabase/migrations/20260615_dispatch_events.sql`
- `POST /api/dispatch/events`
- `GET /api/dispatch/events`

## 검증 지표

기존 배차 대비 AI 우선배차의 효과는 아래 지표로 본다.

- 수락률 증가
- 첫 수락까지 걸린 시간 감소
- `expired` 감소
- `canceled` 감소
- 운행완료수 증가
- 동일 요청수 대비 완료율 증가

### Uber 참고점

- 당장은 기존 배차 후보군을 받아 유사도 정렬에 적용

현재 적용 범위:

- 도착지역 가치와 다음 콜 효율은 후속 버전

## 현재 전달 가능한 결론

기존 배차로직을 바꾸자는 것이 아니다.

콜 발생 시 기존 배차 후보군 안에서 콜카드와 기사 성향의 코사인 유사도가 높은 기사에게 먼저 보내자는 제안이다.

초기 버전은 아래 한 줄로 설명할 수 있다.

```text
기존 후보 기사 목록을 유지하되, 후보 정렬 기준에 22D 코사인 유사도 기반 AI 우선순위를 추가한다.
```

이 방식은 기존 시스템에 대한 변경 부담을 줄이면서도 `expired`, `canceled`, 무응답을 줄이고 운행완료수를 늘릴 수 있는지 검증하기에 적합하다.
