# AI Priority Dispatch Developer Brief

작성일: 2026-06-15
목적: 기존 배차로직에 22D 코사인 유사도 기반 AI 우선순위 정렬을 붙이기 위한 개발 요청 요약

## 한 줄 요청

기존 배차 후보 기사 목록은 유지하되, 콜카드 22D 벡터와 기사 22D 벡터의 코사인 유사도를 계산해 유사도가 높은 기사부터 콜카드를 우선 발송하도록 후보 정렬 기준을 추가한다.

## 왜 필요한가

대시보드에서 요일, 시간, 장소, 거리, 요금 조건에 따라 `expired`, `canceled`가 발생하는 것을 확인했다.

요청수가 증가해도 기사가 수락하지 않으면 운행완료수는 늘지 않는다. 이벤트/프로모션으로 호출을 늘리는 것과 별개로, 실제 운행완료를 늘리려면 콜 조건과 잘 맞는 기사에게 먼저 콜카드를 보내야 한다.

## 구현 범위

이번 요청은 기존 배차 엔진 전체 재작성이 아니다.

개발 범위는 기존 배차 후보군에 AI 우선순위 정렬을 추가하는 것이다.

```text
기존:
콜 발생 -> 기존 후보 기사 탐색 -> 기존 기준 정렬 -> 콜 발송

제안:
콜 발생 -> 기존 후보 기사 탐색 -> 콜카드/기사 22D 코사인 유사도 계산 -> 유사도 높은 순 정렬 -> 콜 발송
```

## 기본 처리 흐름

1. 호출이 발생하면 콜카드 정보를 확보한다.
2. 콜카드 정보를 22차원 콜카드 벡터로 변환한다.
3. 기존 배차로직이 탐색 반경 내 후보 기사 목록을 만든다.
4. 후보 기사들의 `driver_mbti` 22차원 기사 벡터를 조회한다.
5. 콜카드 벡터와 각 기사 벡터의 코사인 유사도를 계산한다.
6. 유사도 높은 순으로 후보를 정렬한다.
7. 상위 기사에게 콜카드를 우선 발송한다.
8. 미수락, 무응답, 만료 시 다음 후보 또는 기존 배차 fallback으로 진행한다.

## 22D 벡터 기준

현재 기준은 `lib/matching-vector.ts`와 동일하게 유지한다.

- 시간대: 새벽, 오전, 주간, 야간
- 요일: 월~일
- 거리: 단거리, 중거리, 장거리
- 요금: 저요금, 중간요금, 고요금
- 콜유형: 유료콜, 무료콜
- 상품: 탄력/할증, 일반
- ETA/근접성

초기 점수:

```text
ai_priority_score = cosine_similarity(call_vector, driver_vector)
```

초기 버전에서는 ETA, 콜 위험도, 예상 수락확률을 최종 점수에 섞지 않는다. 먼저 코사인 유사도 기반 우선 발송이 기존 거리순 대비 효과가 있는지 검증한다.

## 필요한 데이터

### 콜카드

- `callcard_id`
- `asp_id`
- `request_datetime` 또는 `hour_slot`, `weekday`
- `passenger_lat`, `passenger_lng`
- `dest_lat`, `dest_lng`
- `expected_distance`
- `expected_fare`
- `is_paid`
- `is_surge`
- `eta_distance`

### 기사

- `driver_id`
- `vehicle_id`
- `asp_id`
- `driver_mbti` 22D 점수
- 기존 배차 후보군에서 제공하는 현재 상태/거리/반경 정보

## 운영 전 권장 조건

실시간 기사 상태가 가능하면 아래 필터를 기존 후보 생성 단계에 반영한다.

- 온라인 여부
- 공차 여부
- 콜 수신 가능 여부
- 위치 최신성
- 호출 위치 기준 탐색 반경

다만 이 문서의 핵심은 위치 시스템 설계가 아니라, 기존 후보군을 코사인 유사도 기준으로 우선 정렬하는 것이다.

## 결과 저장 권장

효과 검증을 위해 최소한 아래 이벤트는 저장하는 것이 좋다.

- 후보 생성
- 콜카드 발송
- 수락
- 거절
- 무응답
- 만료
- 취소
- 운행완료

저장된 이벤트로 기존 배차 대비 AI 우선배차의 효과를 비교한다.

## 검증 지표

- 수락률 증가
- 첫 수락까지 걸린 시간 감소
- `expired` 감소
- `canceled` 감소
- 운행완료수 증가
- 동일 요청수 대비 완료율 증가

## 전달 파일

- `handoff/AI_PRIORITY_DISPATCH_SPEC.md`: 상세 명세
- `handoff/LIVE_DISPATCH_API_CONTRACT.md`: API/테이블 초안
- `handoff/LIVE_SERVICE_READY_CHECKLIST.md`: 현재 준비 상태와 남은 결정사항
- `lib/matching-vector.ts`: 실제 22D 벡터와 코사인 유사도 기준 코드

## 개발자에게 요청할 결론

기존 배차로직을 교체하지 말고, 기존 배차 후보 기사 목록을 만든 뒤 그 후보들을 22D 코사인 유사도 기준으로 한 번 더 정렬해 달라.

우선순위가 높은 기사에게 먼저 콜카드를 발송하고, 실패하면 기존 방식으로 fallback하면 된다.
