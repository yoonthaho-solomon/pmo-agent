# AI Priority Dispatch Acceptance Test

작성일: 2026-06-15
목적: 개발자가 AI 우선배차 정렬을 구현한 뒤 정상 동작 여부를 검수하기 위한 기준

## 검수 목표

기존 배차 후보군을 유지하면서, 후보 기사 목록이 콜카드 22D 벡터와 기사 22D 벡터의 코사인 유사도 기준으로 우선 정렬되는지 확인한다.

이 테스트는 전체 배차 엔진 교체 여부를 검수하지 않는다. 핵심은 기존 후보 기사 목록에 AI 우선순위 정렬이 안정적으로 추가되었는지 확인하는 것이다.

## 사전 조건

- 콜카드 정보에서 22D 콜카드 벡터를 생성할 수 있어야 한다.
- 후보 기사별 `driver_mbti` 22D 벡터를 조회할 수 있어야 한다.
- 후보 기사 목록은 기존 배차로직 또는 테스트용 후보 생성 로직에서 제공되어야 한다.
- 코사인 유사도 계산식은 `lib/matching-vector.ts`와 동일해야 한다.
- 초기 버전의 최종 AI 우선순위 점수는 `cosine_similarity` 단독으로 검수한다.

## 테스트 1. 콜카드 벡터 생성

### 입력

```json
{
  "hour_slot": 18,
  "weekday": 4,
  "expected_distance": 2500,
  "expected_fare": 9000,
  "is_paid": false,
  "is_surge": false,
  "eta_distance": 240
}
```

### 기대 결과

- 야간 차원 `score_night`가 1이다.
- 금요일 차원 `score_fri`가 1이다.
- 단거리 차원 `score_short`가 1이다.
- 저요금 차원 `score_low_fare`가 1이다.
- 무료콜 차원 `score_free`가 1이다.
- 일반 상품 차원 `score_normal`이 1이다.
- ETA/근접성 `score_near`는 0과 1 사이의 값이다.

## 테스트 2. 기사 벡터 조회

### 입력

기존 배차 후보 기사 목록:

```json
[
  { "driver_id": "DR_A" },
  { "driver_id": "DR_B" },
  { "driver_id": "DR_C" }
]
```

### 기대 결과

- 각 후보 기사에 대해 `driver_mbti` 22D 점수를 조회한다.
- `driver_mbti`가 없는 기사는 AI 정렬 대상에서 제외하거나 기존 정렬 하위로 보낸다.
- 제외/하위 처리 방식은 운영 정책으로 명시한다.

## 테스트 3. 코사인 유사도 정렬

### 조건

동일한 콜카드에 대해 후보 기사 3명의 코사인 유사도가 아래와 같이 계산되었다고 가정한다.

```text
DR_A = 0.82
DR_B = 0.61
DR_C = 0.74
```

### 기대 결과

발송 우선순위는 아래 순서여야 한다.

```text
1. DR_A
2. DR_C
3. DR_B
```

## 테스트 4. 기존 후보군 유지

### 조건

기존 배차로직이 탐색 반경과 기사 상태를 기준으로 후보 20명을 반환한다.

### 기대 결과

- AI 우선배차는 새로운 전체 기사 검색을 임의로 수행하지 않는다.
- 기존 후보 20명 안에서만 코사인 유사도 정렬을 수행한다.
- 기존 후보가 아닌 기사가 AI 추천 결과에 포함되면 실패다.

## 테스트 5. 미수락 fallback

### 조건

AI 우선순위 1위 기사에게 콜카드를 발송했으나 무응답 또는 거절이 발생한다.

### 기대 결과

- 다음 우선순위 기사에게 발송한다.
- 후보가 소진되면 기존 배차 fallback으로 넘어간다.
- fallback 시 기존 배차로직의 안정성을 해치지 않아야 한다.

## 테스트 6. 이벤트 저장

### 조건

AI 우선배차 후보 생성과 발송을 수행한다.

### 기대 결과

최소한 아래 이벤트를 저장할 수 있어야 한다.

- 후보 생성
- 콜카드 발송
- 수락
- 거절
- 무응답
- 만료

각 이벤트에는 가능하면 아래 값이 포함되어야 한다.

- `dispatch_id`
- `callcard_id`
- `driver_id`
- `rank_in_dispatch`
- `vector_cosine`
- `event_type`
- `event_status`
- `event_at`

## 테스트 7. 기존 거리순 대비 비교

### 조건

동일한 콜카드와 후보군에 대해 기존 거리순 결과와 AI 유사도순 결과를 모두 산출한다.

### 기대 결과

아래 비교값을 남긴다.

- 기존 거리순 Top 10
- AI 유사도순 Top 10
- 두 결과의 겹침 수
- AI 1위와 기존 1위가 다른 경우의 이유
- 각 후보의 거리, ETA, 코사인 유사도

## 운영 A/B 테스트 기준

초기 운영 검증은 기존 배차와 AI 우선배차를 비교한다.

권장 지표:

- 수락률
- 첫 수락까지 걸린 시간
- `expired` 비율
- `canceled` 비율
- 운행완료수
- 요청수 대비 완료율

최소 판단 기준:

```text
AI 우선배차가 기존 배차 대비 expired 또는 canceled를 줄이거나,
동일 요청수 대비 운행완료율을 높이면 1차 효과가 있다고 본다.
```

## 실패로 보는 경우

- 기존 후보군 밖의 기사가 AI 결과에 임의로 포함됨
- 콜카드 벡터와 기사 벡터의 차원 순서가 다름
- 코사인 유사도 계산식이 `lib/matching-vector.ts`와 다름
- `final_score`에 검증되지 않은 ETA/위험도/수락확률 가중치가 섞임
- AI 후보 실패 후 기존 배차 fallback이 동작하지 않음
- 발송/수락/거절/만료 이벤트가 남지 않아 사후 분석이 불가능함

## 합격 기준

1. 콜카드 22D 벡터가 정확한 차원 순서로 생성된다.
2. 후보 기사들의 22D 기사 벡터가 정확히 조회된다.
3. 코사인 유사도 계산 결과가 재현 가능하다.
4. 후보 정렬이 유사도 내림차순으로 수행된다.
5. 미수락 또는 무응답 시 다음 후보 또는 기존 fallback이 동작한다.
6. 배차 결과 이벤트가 저장되어 사후 성과 분석이 가능하다.
7. 기존 배차로직을 전면 교체하지 않고 후보 정렬 단계에만 AI 우선순위가 추가된다.
