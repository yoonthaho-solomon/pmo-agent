# PRODUCT SCOPE

## 제품명
KONAMOBILITY — 콜카드·기사 매칭 분석 플랫폼

## 제품 목적
호출·앱미터·기사 운행·매칭 결과 데이터를 연결해 데이터 준비 상태, 벡터 구조, 공간 기반 기사 추천을 한 제품에서 탐색하고 검증한다.

## 핵심 사용자
- 플랫폼사업 운영자
- 배차/매칭 기획자
- 데이터 분석 담당자
- 현장 운영 책임자
- 의사결정자/발표 청중

## 핵심 사용자 흐름

### 흐름 A: 데이터 상태 확인
데이터 운영 → 소스 선택 → 날짜 상태 확인 → 이상 셀 선택 → 상세 원인 확인 → 활용 가능 데이터 확인

### 흐름 B: 벡터 구조 확인
벡터 워크벤치 → 팩터 필터 → 콜카드/기사 비교 → 차이 및 기여도 확인 → 코사인 점수 변화 확인

### 흐름 C: 매칭 실험
매칭 스튜디오 → 조건 입력/지도 선택 → 경로 계산 → 후보 기사 산출 → Top 10 확인 → 기사 선택 → 픽업 경로와 근거 확인

## 범위에 포함
- 3개 작업공간 UI
- 공통 App Shell
- 기존 실데이터 연결
- Mapbox/H3/deck.gl 시각화
- Top 10 랭킹과 상세 근거
- 로딩/빈 상태/오류/부분 적재
- 반응형 데스크톱
- 키보드 focus 및 reduced motion

## 범위에서 제외
- DB 스키마 재설계
- 추천 산식 임의 변경
- 운영 배차 실제 발송
- 관리자 권한 체계 재설계
- 모바일 앱 화면
- 데이터 재적재 배치 작업

## 제품 성공 기준
- 10초 안에 각 화면의 목적을 이해할 수 있다.
- 3개 작업공간이 역할별로 명확히 분리된다.
- 지도는 장식이 아니라 조작 가능한 분석 캔버스다.
- 벡터 비교는 단순 목록이 아니라 차이와 기여도를 설명한다.
- 실제 데이터가 없는 경우 더미 점수로 속이지 않는다.
