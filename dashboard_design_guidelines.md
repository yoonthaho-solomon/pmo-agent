# 행복콜 대시보드 디자인 지침

## 1. 전체 톤
- 다크 관제 대시보드 스타일을 유지한다.
- 배경은 거의 검정에 가까운 `#0b0b0f`, 카드 배경은 `#13131a` 계열을 사용한다.
- 강조색은 제한적으로 사용한다.
  - Red: 위험, 실패, 금지
  - Yellow/Orange: 주의, 피크, 우선순위
  - Green: 안정, 완료, 개선
  - Cyan: 정보, 지도, 보조 강조
- 과한 그라데이션이나 장식 요소는 사용하지 않는다.
- 카드 배경은 차분하게 두고, 색상은 텍스트/라인/태그에만 사용한다.

## 2. 레이아웃 원칙
- 인사이트와 액션플랜은 같은 카드형 구조를 사용한다.
- 화면은 발표용으로 읽히도록 넓고 크게 구성한다.
- 섹션은 세로 스크롤을 허용한다.
- 한 화면에 너무 많은 정보를 빽빽하게 넣지 않는다.
- 핵심 진단 카드 1개 + 상세 카드 여러 개 구조를 기본으로 한다.

## 3. 기본 섹션 구조
권장 구조:

1. 상단 AI 진단 카드
2. 핵심 근거 카드 4개
3. 상세 분석 카드
4. 실행/판단 요약 카드

예시:

```html
<section class="ins-v2-diagnosis critical">
  <div class="ins-v2-diag-kicker">AI 진단</div>
  <div class="ins-v2-diag-main">실패율 37.2% - 노출 후 미수락 구간 점검이 필요합니다</div>
  <div class="ins-v2-diag-sub">EXPIRED, ETA, 노출 기사 수, 승객취소를 함께 봅니다.</div>

  <div class="ins-v2-diag-evidence">
    <div class="ins-v2-diag-card">...</div>
    <div class="ins-v2-diag-card">...</div>
    <div class="ins-v2-diag-card">...</div>
    <div class="ins-v2-diag-card counter">...</div>
  </div>
</section>
```

## 4. 카드 스타일
- 카드 배경: `#13131a`
- 카드 테두리: `1px solid #2a2a30`
- 카드 radius: `14px ~ 18px`
- 카드 padding: `24px ~ 32px`
- 카드 사이 간격: `20px ~ 32px`
- 카드 상단 컬러 라인은 최소화한다.
- 전체 카드가 알록달록해지지 않게 한다.
- 강조는 카드 전체 배경이 아니라: 제목 색 / 숫자 색 / 왼쪽 border / 작은 tag 정도로 제한한다.

## 5. 타이포그래피
- 폰트는 Pretendard 우선.
- 제목은 굵고 크게.
- 본문은 발표 중 읽기 쉽게 짧게.
- 숫자는 크고 강하게.
- 한 카드 안에서 폰트 크기 계층을 명확히 한다.

권장 크기:
- 대진단 제목: `36px ~ 44px`
- 카드 제목: `24px ~ 30px`
- 본문: `18px ~ 22px`
- 핵심 숫자: `48px ~ 72px`
- 보조 설명: `14px ~ 16px`

## 6. 인사이트 카드 패턴
인사이트는 "왜 그런가"를 설명한다.

권장 문장 구조:
- 제목: 짧게
- 진단 연결: 어떤 지표와 연결되는지
- 큰 숫자: 가장 중요한 값
- 보조 설명: 그 숫자를 어떻게 해석해야 하는지
- 비교/순위: TOP3 또는 위험 구간

예시:
```
02
노출 후 미수락 왜 발생했나

진단 연결:
EXPIRED 호출 중 콜카드가 노출된 기사 수가 있었는지 확인합니다.

76.9%
EXPIRED 중 노출 기사 확인 비중

콜카드가 기사에게 노출됐지만 수락되지 않은 호출 비중이 높을수록
단순 공급 부족보다 수락 유도 문제가 클 수 있습니다.
```

## 7. 액션플랜 카드 패턴
액션플랜은 "무엇을 할 것인가"를 보여준다.

권장 구조:
- 하지 말 것
- 먼저 실행할 시간
- 승객 이탈 방어
- 집중 관리 구역

예시:
```html
<div class="ap69-evidence">
  <div class="ap69-ev-card stop">
    <div class="ap69-ev-k">하지 말 것</div>
    <div class="ap69-ev-v">전 지역 일괄 공지 금지</div>
    <div class="ap69-ev-sub">실패가 확인된 시간과 구역부터 좁혀 실행합니다.</div>
  </div>
</div>
```

## 8. 컬러 사용 규칙

| 색상 | 코드 | 용도 |
|------|------|------|
| Red | `#e8271e` | 실패, 위험, 금지, HIGH RISK |
| Yellow | `#ffa602` | 피크, 우선순위, 주의 |
| Green | `#00c49a` | 안정, 완료, 개선 |
| Cyan | `#8eeaff` | 정보, 지도, 보조 강조 |
| Muted text | `#888`, `#9ca3af` | 보조 텍스트 |
| Main text | `#ffffff`, `#e5e7eb` | 본문 |

**사용 금지:**
- 카드마다 서로 다른 배경색을 강하게 넣지 않는다.
- 보라/파랑 그라데이션을 전체 배경으로 깔지 않는다.
- 장식용 오브젝트, 블롭, 과한 그림자는 쓰지 않는다.

## 9. 문장 톤
- 비개발자도 바로 이해할 수 있게 쓴다.
- "계산식"보다 "무엇을 의미하는지"를 먼저 쓴다.
- 단정하지 않는다. 원인으로 확정하지 말고 "가능성", "점검 필요", "우선 확인" 표현을 쓴다.

**좋은 표현:**
- "노출됐지만 수락되지 않은 호출이 많습니다."
- "공급 부족으로 단정하기 전에 노출 기사 수를 함께 봅니다."
- "전체 공지보다 취약 시간대부터 좁혀 실행합니다."

**피할 표현:**
- "기사거부" / "주변차량" / "후보" / "무조건" / "원인은 이것입니다"

## 10. 데이터 용어 기준

| 피할 표현 | 사용할 표현 |
|-----------|-------------|
| 주변차량 | 노출 기사 수 |
| 후보 | 노출 기사 |
| 공급부족 | 노출 기사 없음 |
| 기사거부 | 노출 후 미수락 |
| 콜카드후보 | 액션 검토 |
| grade | 승객 세그먼트 또는 이용자 등급 |
| 미분류 기사 | 미분류 이용자 |

## 11. 계산 기준 문구
툴팁이나 설명에는 아래 기준을 사용한다.

- **전체 호출 수**: 현재 ASP/시간대 필터가 적용된 호출 수
- **성공/완료**: DROP 또는 FINISHED
- **실패율**: (EXPIRED + CANCELED + D_CANCELED) / 전체 호출 수
- **EXPIRED율**: EXPIRED / 전체 호출 수
- **승객취소율**: CANCELED / 전체 호출 수
- **기사취소율**: D_CANCELED / 전체 호출 수
- **노출 기사 수**: idle_car_short_distance_count + idle_car_long_distance_count
- **노출 후 미수락**: EXPIRED 중 콜카드가 1명 이상 기사에게 노출됐지만 수락되지 않은 호출
- **노출 기사 없음**: EXPIRED 중 콜카드가 노출된 기사 수가 0인 호출

## 12. 향후계획 화면 스타일
- 향후계획은 인사이트/액션플랜과 같은 카드 톤을 사용한다.
- 별도 랜딩페이지처럼 만들지 않는다.
- 기존 대시보드 내부 탭처럼 보이게 한다.
- 폭은 인사이트/액션플랜과 유사하게 넓게 사용한다.
- 5줄 요약, 핵심 카드, 다운로드 카드를 같은 카드 시스템 안에 넣는다.

## 13. 반응형 기준
- 기본은 데스크탑/노트북/빔프로젝터 기준.
- 모바일 최적화보다 발표 가독성을 우선한다.
- 1100px 이하에서만 단일 컬럼 전환을 허용한다.
- 카드 텍스트가 잘리는 것보다 세로로 길어지는 것을 허용한다.

## 14. 수정 원칙
- 기존 업로드, KPI, 지도, H3, Mapbox 로직은 건드리지 않는다.
- 디자인 수정은 가능한 한 CSS와 렌더링 후처리로 제한한다.
- 지도와 분석모드가 안정화된 파일을 기준으로 작업한다.
- 큰 구조 변경보다 카드/문구/간격/타이포그래피를 정리한다.
